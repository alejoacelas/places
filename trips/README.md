# trips

Getting from one place to another, optimized.

The dream: Google Maps but better. Plan a journey, compare driving vs train vs
multimodal on the things I actually care about, and — the fun part — buy the
ticket straight from my terminal and carry the whole thing on my phone.

Right now this is a plan, not code. The routing half looks very doable without
locking into Google. The ticket-buying half is the hard part, and the plan is
honest about how far it really goes.

Start at [plan/00-overview.md](plan/00-overview.md). The raw derisking lives in
[plan/research/](plan/research/).
